## 关键实现说明

跨聚合推进顺序固定为 `Review → Shot → Pipeline`。Review 事件先通过
`ApplyShotReviewResult` 命令改变 Shot；随后 Shot 事件再完成或失败 Pipeline
审核节点，避免同一业务结果由 Review 和 Shot 两条路径重复推进 Pipeline。
事件 ID 派生为稳定 command ID，重复投递返回幂等成功。
