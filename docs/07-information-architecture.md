# 第三章：系统信息架构设计（IA）

## 首页工作台设计原则

首页采用：

> **A 项目驾驶舱 + B AI创作工作台 + C AI生产流水线监控 三合一**

这个设计符合产品定位：

> **AI内容生产操作系统**

因为不同角色进入系统，关注的信息完全不同：

* 项目负责人关心：项目是否按计划推进
* 创作者关心：我今天要做什么、AI怎么帮我完成
* 管理员关心：模型、成本、任务队列是否正常

因此首页采用**角色化工作台（Role-based Dashboard）**设计。

---

## 信息架构设计原则

### 1. 用户角色驱动

根据不同用户角色（项目负责人、创作者、管理员）设计不同的信息展示和操作入口。

### 2. 任务流程驱动

围绕生产流程（剧本→分镜→图片→视频→审核→发布）组织信息结构。

### 3. 资产优先原则

资产中心作为核心竞争力，提供独立的资产管理和复用入口。

### 4. AI能力集成

创作中心和AI生产中心体现AI时代的产品特点，将AI能力深度集成到工作流程中。

### 5. 数据驱动决策

数据中心提供全面的数据分析和决策支持能力。

### 6. 可扩展性设计

信息架构支持后续功能扩展（如AI小说、AI漫画、数字人等）。

---

## 信息架构特点

### 核心特点

1. **角色化工作台**：首页根据用户角色动态展示不同内容
2. **项目驱动**：项目中心作为业务主入口，贯穿整个生产流程
3. **创作中心**：独立创作中心，体现AI时代的创作特点
4. **AI生产中心**：专门的AI能力调用和管理中心
5. **资产中心**：独立的资产管理和复用体系
6. **数据驱动**：完整的数据分析和决策支持体系

### 与传统项目管理的区别

- **新增创作中心**：区别于传统项目管理，专注创作本身
- **新增AI生产中心**：管理AI能力和模型调用
- **强化资产中心**：资产复用是AI生产的核心竞争力
- **强化数据中心**：AI成本和质量数据需要特别关注

### 与AI工具的区别

- **项目管理**：不仅是生成工具，而是完整的项目管理系统
- **流程管理**：支持完整的生产流程管理，不只是单点生成
- **团队协作**：支持团队协作和权限管理
- **资产沉淀**：所有成果沉淀为资产，持续复用价值
