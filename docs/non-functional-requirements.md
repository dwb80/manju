# 非功能性需求文档（Non-Functional Requirements）

> 本文档定义《AI漫剧项目管理系统》V1.0的非功能性需求(NFR),包括性能、安全、可用性、可扩展性等方面。

---

## 1. 性能需求（Performance Requirements）

### 1.1 响应时间

| 操作类型 | 响应时间要求 | 备注 |
|---------|------------|------|
| **页面加载** | | |
| 首页加载(初始) | ≤ 2秒 | 首屏渲染完成 |
| 项目列表页加载 | ≤ 1秒 | 含20个项目 |
| 项目详情页加载 | ≤ 1.5秒 | 含剧集、分镜列表 |
| **用户交互** | | |
| 按钮点击响应 | ≤ 200ms | 立即反馈 |
| 表单提交响应 | ≤ 500ms | 含前端验证 |
| 搜索/筛选响应 | ≤ 1秒 | 含1000条数据 |
| **API请求** | | |
| 数据查询API | ≤ 500ms | 单表查询 |
| 数据写入API | ≤ 1秒 | 含数据库事务 |
| 批量操作API | ≤ 3秒 | 批量更新50条 |
| **AI生成** | | |
| 文本生成(剧本分析) | ≤ 30秒 | 或60秒超时 |
| 图片生成(单张) | ≤ 30秒 | 4张候选图 |
| 视频生成(单个) | ≤ 5分钟 | 或异步处理 |

**验证方法**:
- 使用性能测试工具(如JMeter/Lighthouse)
- 监控实际响应时间(P95/P99指标)
- 设置性能告警阈值

---

### 1.2 并发处理能力

| 场景 | 并发用户数 | 性能要求 |
|------|----------|---------|
| **正常负载** | 50用户 | 响应时间正常,无卡顿 |
| **高峰负载** | 200用户 | 响应时间可增加50% |
| **极端负载** | 500用户 | 系统稳定,允许降级服务 |
| **AI生成队列** | 100任务 | 队列不阻塞,优先级调度 |

**并发策略**:
- 使用连接池(数据库/API连接)
- 实现异步任务队列(Bull/Redis)
- 采用负载均衡(多实例部署)
- 实施请求限流(防止滥用)

---

## 2. 安全需求（Security Requirements）

### 2.1 身份认证与访问控制

**认证机制**:
- [ ] 用户登录认证(JWT Token)
- [ ] Token过期时间:24小时
- [ ] 支持刷新Token(有效期7天)
- [ ] 密码加密存储(bcrypt/加盐)
- [ ] 密码强度要求:至少8位+大小写+数字
- [ ] 支持多因素认证(MFA,V2可选)

**权限控制**:
- [ ] 基于角色的访问控制(RBAC)
- [ ] 项目级权限隔离(只能访问自己项目)
- [ ] API权限验证(每个接口验证Token)
- [ ] 操作日志记录(记录所有关键操作)
- [ ] 权限变更审计(记录权限分配历史)

**会话管理**:
- [ ] 单用户最多5个并发Session
- [ ] 超时自动登出(30分钟无操作)
- [ ] 异地登录检测与通知
- [ ] 支持强制登出(管理员操作)

---

### 2.2 数据安全

**数据传输安全**:
- [ ] 所有API使用HTTPS(TLS 1.2+)
- [ ] 禁止明文传输敏感信息
- [ ] API请求签名验证(防篡改)
- [ ] CORS白名单限制

**数据存储安全**:
- [ ] 数据库密码加密存储
- [ ] 敏感字段加密(如手机号)
- [ ] 文件存储权限控制(私有访问)
- [ ] 定期数据备份(每日备份)
- [ ] 备份数据加密存储

**数据完整性**:
- [ ] 关键数据操作使用数据库事务
- [ ] 数据版本控制(剧本/分镜)
- [ ] 数据变更记录(审计日志)
- [ ] 数据恢复机制(从备份恢复)

---

### 2.3 防护机制

**常见攻击防护**:
- [ ] SQL注入防护(参数化查询)
- [ ] XSS攻击防护(输入过滤+输出编码)
- [ ] CSRF防护(Token验证)
- [ ] 文件上传漏洞防护(文件类型验证+重命名)
- [ ] API滥用防护(请求频率限制)

**API安全**:
- [ ] 请求频率限制(单用户:100次/分钟)
- [ ] IP黑名单机制
- [ ] 异常请求告警
- [ ] API调用日志记录

**AI服务安全**:
- [ ] AI API密钥加密存储
- [ ] AI调用参数审计
- [ ] AI生成内容过滤(敏感词检测)
- [ ] 防止Prompt注入攻击

---

### 2.4 内容安全

**内容审核**:
- [ ] AI生成内容自动审核(关键词/图像识别)
- [ ] 支持人工审核流程
- [ ] 内容合规性检查(政治/色情/暴力)
- [ ] 审核记录留存(6个月)

**版权保护**:
- [ ] 素材版权标记(来源/授权信息)
- [ ] AI生成内容版权声明
- [ ] 用户协议明确版权归属
- [ ] 版权争议处理流程

## 3. 可用性需求（Availability Requirements）

### 3.1 系统可用性

| 指标 | 目标值 | 说明 |
|------|-------|------|
| **系统可用率** | 99.5% | 每月停机时间≤3.6小时 |
| **计划停机** | ≤ 2小时/月 | 维护升级时间 |
| **非计划停机** | ≤ 1.6小时/月 | 故障恢复时间 |
| **故障恢复时间(MTTR)** | ≤ 30分钟 | 从故障到恢复 |
| **数据恢复时间** | ≤ 4小时 | 从备份恢复完整数据 |

---

### 3.2 容错机制

**数据容错**:
- [ ] 数据库主从备份
- [ ] 自动故障切换(主库故障切换到备库)
- [ ] 数据备份恢复测试(每月一次)
- [ ] 关键数据操作失败重试(3次)

**服务容错**:
- [ ] 服务健康检查(每5分钟)
- [ ] 服务异常自动重启
- [ ] 多实例部署(至少2个实例)
- [ ] 负载均衡器健康检查

**AI服务容错**:
- [ ] AI API调用失败重试(最多3次)
- [ ] AI服务降级策略(默认模型不可用时切换备用模型)
- [ ] AI队列管理(防止队列阻塞)
- [ ] AI任务超时处理(30秒文本/5分钟视频)

---

### 3.3 备份与恢复

**备份策略**:
| 备份类型 | 频率 | 保留时长 | 存储位置 |
|---------|------|---------|---------|
| 数据库全量备份 | 每日 | 30天 | 云存储(加密) |
| 数据库增量备份 | 每小时 | 7天 | 云存储 |
| 文件资产备份 | 每周 | 90天 | 云存储 |
| 配置文件备份 | 每次变更 | 永久 | Git仓库 |

**恢复测试**:
- [ ] 每月执行一次备份恢复演练
- [ ] 恢复时间验证(≤4小时)
- [ ] 数据完整性验证(抽样检查)
- [ ] 恢复操作文档化

## 附录:NFR优先级定义

### P0(MVP必须满足)
- 性能:响应时间、并发能力
- 安全:身份认证、数据安全、内容审核
- 可用性:99.5%可用率、容错机制
- 易用性:基础交互、错误处理

### P1(重要但可延后)
- 性能:数据容量优化、资源监控
- 安全:多因素认证、高级防护
