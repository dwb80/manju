## 一、核心概念澄清（重要更正）

### 1.1 正确的角色工厂和场景工厂定义

**角色工厂** = 生成角色图片的**生产车间**  
**场景工厂** = 生成场景图片的**生产车间**

它们不是资产库，而是**生产模块**！

```
AI生产中心
├── 图片生成（通用）
├── 视频生成（通用）
├── 角色工厂（专门生成角色图片）⭐ 生产车间
└── 场景工厂（专门生成场景图片）⭐ 生产车间

资产中心（仓库）
├── 角色资产（来自角色工厂的筛选结果）
└── 场景资产（来自场景工厂的筛选结果）
```

## 二、角色工厂的工作流程

### 2.1 角色工厂功能定位

**角色工厂**是一个专门生成角色图片的生产车间，核心功能：

| 功能 | 说明 |
|------|------|
| **角色图片生成** | 基于角色描述生成多张角色参考图 |
| **一致性优化** | 支持图生图，保持角色一致性 |
| **批量生成** | 一次生成多张不同角度、表情的角色图 |
| **临时存储** | 生成的图片存储在ImageTask表中 |
| **用户筛选** | 提供界面让用户选择可用的图片 |

---

### 2.2 角色工厂工作流程

```
1. 用户输入角色描述
   ↓
2. AI生成多张角色图片（3-5张）
   ↓
3. 保存到临时存储区（ImageTask表）
   ├── task_id: img-001
   ├── prompt: "年轻男性，长发，白衣，少年感..."
   ├── image_urls: ["角色1.png", "角色2.png", "角色3.png"]
   └── status: completed
   ↓
4. 用户查看生成的图片
   ↓
5. 用户评估图片质量
   ├── 角色特征是否准确？
   ├── 一致性是否良好？
   └── 是否符合剧本设定？
   ↓
6. 用户选择可用的图片
   ├── 标记为"可用"
   └── 或重新生成
   ↓
7. 保存到资产中心
   ├── 创建角色资产
   ├── 填写角色名称
   ├── 保存角色描述（prompt）
   ├── 保存选中的图片URL
   └── 绑定到项目
```

## 四、资产中心的入库机制

### 4.1 资产保存流程

当用户在角色工厂或场景工厂中选择可用的图片后，点击"保存为资产"：

```typescript
// 保存角色资产
async function saveCharacterAsset(selectedImageUrl: string, characterData: any) {
  const asset = {
    project_id: currentProjectId,
    kind: "character",
    name: characterData.name,              // 角色名称
    prompt: characterData.prompt,          // 角色描述
    image_url: selectedImageUrl,           // 用户选择的图片URL
    role_traits: characterData.traits,     // 角色特征
    role_images: [selectedImageUrl],       // 参考图片列表
    tags: ["主角", "第一集"],              // 标签
    notes: characterData.notes,            // 备注
  };
  
  await createAsset(currentProjectId, asset);
}

// 保存场景资产
async function saveSceneAsset(selectedImageUrl: string, sceneData: any) {
  const asset = {
    project_id: currentProjectId,
    kind: "scene",
    name: sceneData.name,                  // 场景名称
    prompt: sceneData.prompt,              // 场景描述
    image_url: selectedImageUrl,           // 用户选择的图片URL
    style_keywords: sceneData.styleKeywords, // 风格关键词
    tags: ["室内", "第一集"],              // 标签
    notes: sceneData.notes,                // 备注
  };
  
  await createAsset(currentProjectId, asset);
}
```

## 五、剧本中心的资产引用

### 5.1 引用流程

```
剧本中心
├── Scene01: 茶信馆
│   ├── 场景引用：scene_asset_id: 25（茶信馆资产）
│   ├── 角色引用：character_asset_id: 12（林逸资产）
│   └── 角色引用：character_asset_id: 15（萧晓资产）
│
└── 数据库记录
    ├── script_scene_location表
    │   ├── scene_id: 101
    │   └── location_asset_id: 25
    │
    └── script_scene_character表
        ├── scene_id: 101
        ├── character_asset_id: 12
        └── character_asset_id: 15
```

---

### 5.2 引用关系总结

```
角色工厂（生产）→ 用户筛选 → 资产中心（仓库）→ 剧本中心（引用）
场景工厂（生产）→ 用户筛选 → 资产中心（仓库）→ 剧本中心（引用）
```

**关键点**:
- ✅ 角色工厂和场景工厂是**生产车间**，生成多张图片供用户选择
- ✅ 用户选择后的图片才保存到资产中心，成为**角色资产**或**场景资产**
- ✅ 剧本中心引用的是资产中心的**已入库资产**
- ✅ 所有模块必须绑定**同一个项目**

## 六、项目绑定关系（确认）

### 6.1 项目绑定规则

| 模块 | 项目绑定 | 说明 |
|------|---------|------|
| **角色工厂** | ✅ 必须绑定项目 | 生成的图片属于特定项目 |
| **场景工厂** | ✅ 必须绑定项目 | 生成的图片属于特定项目 |
| **资产中心** | ✅ 必须绑定项目 | 资产属于特定项目 |
| **剧本中心** | ✅ 必须绑定项目 | 剧本属于特定项目 |
| **引用关系** | ✅ 必须同一项目 | 剧本只能引用同一项目的资产 |

---

### 6.2 数据流转路径

```
项目A (proj-001)
├── 角色工厂
│   ├── 生成角色图片 → ImageTask (project_id: proj-001)
│   └── 用户选择 → 保存到资产中心
│
├── 资产中心
│   ├── 角色资产ID: 12（project_id: proj-001）
│   └── 场景资产ID: 25（project_id: proj-001）
│
└── 剧本中心
    ├── Scene01（project_id: proj-001）
    ├── 引用角色资产ID: 12（同一项目）✅
    └── 引用场景资产ID: 25（同一项目）✅

❌ 不允许：剧本引用其他项目的资产
```

## 七、临时存储区的管理

### 7.1 ImageTask表的作用

```sql
CREATE TABLE image_tasks (
    id TEXT PRIMARY KEY,
    conversation_id TEXT,       -- 会话ID（生成任务上下文）
    prompt TEXT NOT NULL,       -- 提示词
    image_urls TEXT,            -- 生成的图片URL列表（JSON）
    status TEXT,                -- 任务状态
    error TEXT,                 -- 错误信息
    created_at TEXT             -- 创建时间
);
```

**作用**:
- ✅ 临时存储生成的图片
- ✅ 记录生成历史
- ✅ 支持用户查看和选择
- ✅ 选择后可删除或归档

---

### 7.2 临时存储生命周期

```
1. 角色工厂生成图片
   ↓
2. 保存到ImageTask表
   ├── status: completed
   └── image_urls: ["角色1.png", "角色2.png", "角色3.png"]
   ↓
3. 用户查看和选择
   ↓
4. 用户选择可用的图片
   ↓
5. 保存到资产中心
   ↓
6. 删除或归档ImageTask记录
   ├── 保留7天（供用户回顾）
   └── 7天后自动清理
```

## 八、总结

### 核心概念（重要）

| 概念 | 定义 | 作用 |
|------|------|------|
| **角色工厂** | 生产车间 | 生成多张角色图片供用户选择 |
| **场景工厂** | 生产车间 | 生成多张场景图片供用户选择 |
| **资产中心** | 仓库 | 存储用户选择的可用图片（角色资产、场景资产） |
| **剧本中心** | 使用者 | 引用资产中心的角色资产和场景资产 |

---

### 数据流转路径

```
生产 → 筛选 → 入库 → 使用

角色工厂 → 用户选择 → 资产中心（角色资产）→ 剧本中心引用
场景工厂 → 用户选择 → 资产中心（场景资产）→ 剧本中心引用
```

---

### 项目绑定规则

- ✅ 所有模块必须绑定同一个项目
- ✅ 剧本只能引用同一项目的资产
- ✅ 跨项目资产引用会导致数据混乱和权限问题
