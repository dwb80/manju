## 角色权限设计原则

不是简单：

```
用户 → 菜单权限
```

而是：

```
用户

↓

角色

↓

项目权限

↓

资源权限

↓

操作权限
```

例如：

一个美术人员：

项目A：

可以：

✅ 查看角色资产
✅ 创建图片任务
✅ 修改Prompt

项目B：

只能：

✅ 查看

---

## 核心设计要点

### 1. 项目级权限隔离

用户在不同项目中可能有不同的权限级别，确保项目间的数据和资源隔离。

### 2. 资源级操作权限

除了查看权限，还需要定义创建、编辑、删除等操作权限，确保资源的安全性。

### 3. 角色继承与组合

允许一个用户同时拥有多个角色，权限自动叠加。

### 4. 权限动态调整

支持项目负责人临时调整成员权限，适应项目不同阶段的需求。

## 权限管理最佳实践

### 1. 最小权限原则

只授予用户完成工作所需的最小权限，避免过度授权。

### 2. 权限分离原则

关键操作需要多个角色共同确认，避免单点风险。

### 3. 权限审计原则

定期审计权限分配，及时清理不必要的权限。

### 4. 权限继承原则

合理设计角色层级，利用继承机制简化权限管理。
