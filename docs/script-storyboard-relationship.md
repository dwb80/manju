## 二、剧本中心与分镜导演台的关系

### 2.1 关系定义

| 概念 | 定义 | 职责 |
|------|------|------|
| **剧本中心** | 剧本创作阶段 | 创作、编辑、管理剧本内容（文字） |
| **分镜导演台** | 分镜设计阶段 | 将剧本拆分为分镜、设计镜头（视觉） |
| **关系** | 剧本提供内容，分镜进行拆分 | 剧本中心 → 分镜导演台 |

---

### 2.2 数据流转方向

```
剧本中心（输出）
├── 剧本内容（文字）
├── 角色设定（文字描述）
├── 场景设定（文字描述）
├── 对白内容
└── 剧情结构

    ↓ 提供内容

分镜导演台（输入）
├── 读取剧本内容
├── 拆分为多个分镜
├── 设计镜头参数
├── 分配角色和场景
└── 生成分镜图片
```

### 3.2 数据库关联设计

#### project_storyboards 表（分镜表）

```sql
CREATE TABLE project_storyboards (
    id                  TEXT PRIMARY KEY,
    project_id          TEXT NOT NULL,              -- 项目绑定 ⭐
    
    -- 剧本关联字段
    episode             INTEGER,                    -- 剧集号
    scene               TEXT,                       -- 场景名称
    shot                TEXT,                       -- 镜头编号
    
    -- 内容字段
    title               TEXT,                       -- 分镜标题
    description         TEXT,                       -- 分镜描述
    dialogue            TEXT,                       -- 对白内容
    
    -- 资产引用字段 ⭐ 关键
    characters          TEXT,                       -- 角色名称列表（JSON）
    character_asset_ids TEXT,                       -- 角色资产ID列表（JSON）⭐
    location            TEXT,                       -- 地点名称
    scene_asset_id      TEXT,                       -- 场景资产ID ⭐
    
    -- 镜头参数
    shot_size           TEXT,                       -- 景别
    camera_move         TEXT,                       -- 镜头运动
    duration            INTEGER,                    -- 时长（秒）
    
    -- AI生成字段
    prompt              TEXT,                       -- 提示词
    image_task_id       TEXT,                       -- 图片任务ID
    image_url           TEXT,                       -- 图片URL
    video_task_id       TEXT,                       -- 视频任务ID
    video_url           TEXT,                       -- 视频URL
    
    -- 状态字段
    status              TEXT DEFAULT 'draft',       -- 状态
    notes               TEXT,                       -- 备注
    
    -- 时间字段
    created_at          TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at          TEXT DEFAULT CURRENT_TIMESTAMP,
    
    -- 索引
    INDEX idx_project_episode (project_id, episode),
    INDEX idx_scene_asset (scene_asset_id),
    INDEX idx_status (status)
);
```

---

### 3.3 关联字段说明

| 字段 | 关联对象 | 说明 |
|------|---------|------|
| **character_asset_ids** | 角色资产 | 引用角色资产ID列表（如 ["role-12", "role-15"]） |
| **scene_asset_id** | 场景资产 | 引用场景资产ID（如 "scene-25"） |
| **project_id** | 项目 | 剧本和分镜必须绑定同一项目 |

## 四、分镜拆分流程

### 4.1 手动拆分流程

```
1. 用户在剧本中心完成剧本创作
   ├── Scene01: 茶信馆
   │   ├── 林逸：走了出来
   │   └── 萧晓：终于舍得出来了
   └── Scene02: 长街
       └── 林逸：与你无关

    ↓

2. 用户进入分镜导演台
   ├── 选择剧本（从剧本中心）
   └── 查看剧本内容

    ↓

3. 用户手动拆分分镜
   ├── Scene01拆分为3个镜头
   │   ├── 镜头1-1：林逸推开木门（中景）
   │   ├── 镜头1-2：林逸走进茶信馆（跟镜头）
   │   └── 镜头1-3：萧晓抬头看林逸（特写）
   └── Scene02拆分为2个镜头
       ├── 镜头2-1：林逸走在长街上（远景）
       └── 镜头2-2：林逸停下脚步（中景）

    ↓

4. 用户填写分镜参数
   ├── 角色：引用角色资产ID
   ├── 场景：引用场景资产ID
   ├── 景别：远景/中景/近景/特写
   ├── 运动：推/拉/摇/移/跟
   └── 时长：3秒/5秒/10秒

    ↓

5. 系统自动生成提示词
   ├── 基于角色资产：角色外观、服装、特征
   ├── 基于场景资产：场景氛围、光影、色调
   └── 基于分镜描述：动作、对白、镜头运动

    ↓

6. AI生成分镜图片
   ├── 调用图片生成API
   └── 保存图片URL
```

---

### 4.2 AI自动拆分流程

```
1. 用户在剧本中心完成剧本创作

    ↓

2. 用户点击【AI拆分分镜】按钮

    ↓

3. AI读取剧本内容
   ├── 解析剧本结构
   ├── 提取场景、角色、对白
   └── 理解剧情节奏

    ↓

4. AI自动拆分分镜
   ├── 分析场景节奏
   ├── 确定镜头数量
   ├── 设计镜头参数
   └── 分配角色和场景

    ↓

5. AI生成分镜列表
   ├── 镜头1-1：远景，推镜头，5秒
   ├── 镜头1-2：中景，跟镜头，3秒
   ├── 镜头1-3：近景，固定镜头，2秒
   └── ...

    ↓

6. 用户审核和调整
   ├── 查看AI生成的分镜列表
   ├── 调整镜头参数
   ├── 修改角色和场景引用
   └── 确认分镜列表

    ↓

7. AI生成分镜图片
   ├── 批量生成分镜图片
   └── 保存图片URL
```

## 六、项目绑定关系（重要）

### 6.1 是否必须绑定同一个项目？

**答案**: ✅ **必须绑定同一个项目**

---

### 6.2 为什么必须绑定同一个项目？

#### 原因1: 数据隔离

```
项目A (proj-001)
├── 剧本中心
│   ├── Scene01: 茶信馆
│   └── Scene02: 长街
│
└── 分镜导演台
    ├── 镜头1-1（引用Scene01）
    ├── 镜头1-2（引用Scene01）
    └── 镜头2-1（引用Scene02）

项目B (proj-002)
├── 剧本中心
│   └── Scene01: 客栈
│
└── 分镜导演台
    └── 镜头1-1（引用Scene01）
```

**如果允许跨项目引用**:
- ❌ 项目A的分镜引用项目B的剧本 → 数据混乱
- ❌ 项目删除时，分镜引用关系复杂，难以清理
- ❌ 权限控制困难，跨项目访问剧本不安全

---

#### 原因2: 权限控制

```
用户角色权限：
- 导演：可访问项目A的所有剧本和分镜
- 编剧：只能访问项目A的剧本
- 分镜师：只能访问项目A的分镜

如果允许跨项目引用：
- ❌ 分镜师可能引用项目B的剧本（权限越界）
- ❌ 编剧无法理解项目A的分镜为什么引用项目B的剧本
```

---

#### 原因3: 数据迁移

```
项目导出：
- 导出项目A的分镜 → 需要同时导出项目A的剧本、角色、场景资产
- 如果引用了项目B的剧本 → 导出失败或数据不完整

项目删除：
- 删除项目A → 需要清理项目A的所有分镜引用
- 如果有其他项目引用项目A的剧本 → 删除失败
```

---

### 6.3 项目绑定规则

| 模块 | 项目绑定 | 说明 |
|------|---------|------|
| **剧本中心** | ✅ 必须绑定项目 | 剧本属于特定项目 |
| **角色资产** | ✅ 必须绑定项目 | 角色属于特定项目 |
| **场景资产** | ✅ 必须绑定项目 | 场景属于特定项目 |
| **分镜导演台** | ✅ 必须绑定项目 | 分镜属于特定项目 |
| **关联关系** | ✅ 必须同一项目 | 分镜只能引用同一项目的剧本和资产 |

---

### 6.4 数据库约束设计

```sql
-- 分镜表
CREATE TABLE project_storyboards (
    id              TEXT PRIMARY KEY,
    project_id      TEXT NOT NULL,  -- 必须绑定项目
    -- ...
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 分镜引用角色资产
-- ⭐ 确保角色资产和分镜属于同一项目
CHECK (
    (SELECT project_id FROM project_storyboards WHERE id = storyboard_id) =
    (SELECT project_id FROM project_assets WHERE id = character_asset_id)
);

-- 分镜引用场景资产
-- ⭐ 确保场景资产和分镜属于同一项目
CHECK (
    (SELECT project_id FROM project_storyboards WHERE id = storyboard_id) =
    (SELECT project_id FROM project_assets WHERE id = scene_asset_id)
);
```

## 九、总结

### 核心要点

1. **一个剧本可以拆分出多个角色、多个场景、多个分镜** ✅ 完全正确
   - 一个剧本 → 多个角色设定
   - 一个剧本 → 多个场景设定
   - 一个剧本 → 多个分镜

2. **剧本中心与分镜导演台的关系**
   - 剧本中心：提供剧本内容（文字）
   - 分镜导演台：拆分剧本为分镜（镜头）
   - 关联方式：通过 `character_asset_ids` 和 `scene_asset_id` 引用资产

3. **必须绑定同一个项目** ✅ 完全正确
   - 剧本、角色资产、场景资产、分镜都必须绑定同一项目
   - 确保数据隔离、权限控制、数据迁移的安全性

---

### 数据流转路径

```
剧本中心 → 资产中心 → 分镜导演台 → 视频中心
   ↓         ↓         ↓           ↓
剧本文字   角色场景图片   分镜图片     视频片段
```

---

### 关键设计原则

| 原则 | 说明 |
|------|------|
| ✅ **剧本拆分** | 一个剧本可拆分为多个分镜 |
| ✅ **资产引用** | 分镜通过ID引用角色资产和场景资产 |
| ✅ **项目绑定** | 所有模块必须绑定同一项目 |
| ✅ **数据隔离** | 不同项目的数据完全独立 |
| ✅ **流程清晰** | 剧本 → 资产 → 分镜 → 视频 |
