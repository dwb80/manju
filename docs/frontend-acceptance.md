# 前端页面验收标准

本文档用于 Agnes AI Studio 前端交付验收。优先级分为：

- P0：上线阻断。白屏、核心功能不可用、数据串会话、保存失败、误删、导出不可用都属于 P0。
- P1：重要体验问题。布局错位、空状态缺失、筛选不准、错误提示不清楚、移动端难用属于 P1。
- P2：体验优化。动效、微文案、视觉细节、非核心兼容性属于 P2。

## 通用验收

### 视觉与布局

P0：
- 页面不能出现内容重叠、主要按钮不可见、输入框遮挡操作区。
- 图片和视频必须按原比例展示，不允许拉伸变形。
- 320px 到 1920px 宽度下核心页面可用，不出现破坏性横向滚动。
- 文本不能撑破按钮、卡片、菜单和表格单元格。

P1：
- 同类组件的间距、圆角、边框、阴影保持一致。
- 深色背景下文字对比度必须可读。
- 空数据必须展示空状态说明，不能只留空白。
- hover、active、disabled 状态要有明确反馈。

### 交互

P0：
- 所有可点击按钮必须有有效结果，不能点击无反应。
- 保存、删除、导出、生成、上传必须有成功或失败反馈。
- 请求中的按钮要防重复提交。
- 删除、移除、归档等危险操作必须二次确认。

P1：
- 输入框聚焦态清晰，选择框和筛选条件能即时更新。
- 搜索、筛选、重置逻辑准确。
- 图片和视频详情按需求在新标签页打开。
- 底部输入框不遮挡历史消息或生成结果。

### 功能逻辑

P0：
- 查询、提交、保存、删除、导出接口调用成功。
- 接口失败或网络失败时页面不白屏，展示明确错误。
- 会话、图片、视频、项目、资产必须绑定正确归属，不能串数据。
- 删除会话时，关联消息、图片任务、视频任务、收藏记录按需求删除。
- 删除项目时，项目下团队、剧集、问题、里程碑、任务、剧本、分镜、剪辑、审核、资产记录按需求处理。

P1：
- 列表支持关键词、状态、负责人等筛选时，结果要准确。
- 表单必填项、数字范围和格式限制要生效。
- 加载中、无结果、失败状态都要有兜底展示。

### 工程

P0：
- `npm run build` 必须通过。
- 后端相关改动必须通过 `npm test`。
- 不能留下导致构建失败的类型错误。
- 不直接渲染用户输入的原生 HTML。

P1：
- 接口调用统一封装，错误统一处理。
- 可点击元素优先使用 `button`，图片要有 `alt`。
- 新增复杂模块应拆成独立文件，避免继续堆到超长页面组件里。

## Agnes AI Studio 专项验收

### 聊天

P0：
- 新建会话后可以发送消息。
- 一个会话正在生成时，新建其他会话不能中断原任务，也不能导致新会话无法发送。
- 停止生成、重新生成入口可用，重新生成不能重复插入用户消息。
- 点击历史会话后，消息展示区显示对应会话内容。

P1：
- 历史会话标题使用第一个用户问题。
- 图片附件发送后要进入消息展示区，不继续停留在输入框。
- 聊天、图片、视频模式切换不丢当前会话归属。

### 图片生成

P0：
- 支持多张图片附件上传和预览。
- 生成结果绑定当前会话和当前项目。
- 历史会话中的图片只显示在所属会话下。
- 图片详情页能新标签页打开，并支持查看、下载、复制、收藏、删除。
- 图片按原比例展示，9:16 图片按竖图比例显示。

P1：
- 生成中有 loading。
- 失败时显示 Agnes 错误或本地错误提示。
- 参考图可继续编辑，可加入项目资产库。

### 视频生成

P0：
- 视频生成任务绑定当前会话和项目。
- 视频结果可播放、打开详情、下载。
- 分镜底图资产可以直接带入图生视频。
- Agnes 429 等限流错误要提示清楚，不能无限 loading。

P1：
- 视频输入框附件发送后进入信息展示区。
- 视频预览不遮挡输入区。
- 状态轮询频率不能过高，避免触发查询限流。

### 项目与历史会话

P0：
- 历史会话按项目归属分组展示。
- 点击项目下的会话，展示对应聊天、图片或视频信息。
- 项目菜单中的新建会话、置顶、打开目录、重命名、归档、移除功能可用。

P1：
- 新建项目时可设置存储文件夹。
- 创建新会话后，其他项目和会话不能被隐藏。

### 项目工作台

P0：
- 项目首页、团队、剧集、问题、里程碑、剧本、分镜、剪辑、审核、任务、资产、交付工作页可正常切换。
- 项目首页可进入剧本、分镜、资产、剪辑、审核、交付等独立工作页。
- 工作台 URL 深链可刷新恢复，浏览器前进后退能同步项目和工作页。
- 新增、更新、删除操作调用接口成功。
- 项目不存在时要退出坏 ID，不能持续报 `project not found`。

P1：
- 工作台筛选支持关键词、状态、负责人，并能重置。
- 项目健康度能根据问题、里程碑、分镜、资产、审核、任务给出合理提示。
- 管理型页面采用表格列表加标准新增/编辑表单。

### 资产库

P0：
- 支持图片、视频、角色、场景、风格、提示词、项目资料、分镜底图分类。
- 资产可新增、删除、收藏、取消收藏。
- 图片和视频资产可作为生成输入源复用。
- 角色和场景资产可绑定到分镜。

P1：
- 支持关键词、标签、类型、收藏筛选。
- 复制提示词、设为参考图、用于视频等按钮可用。

### 交付导出

P0：
- 剧本文档 TXT、分镜表 CSV、剪辑清单 CSV、项目清单 JSON 可导出。
- CSV 中文不乱码。
- 交付包索引可生成到项目本地 `exports` 目录。

## 验收命令

后端：

```bat
cd backend
npm test
```

前端：

```bat
cd frontend
npm run build
npm run test:e2e
```

手工验收建议：

1. 启动 `start-all.bat`。
2. 用 Chrome 或 Edge 打开 `http://localhost:3001`。
3. 先检查所有 P0。
4. P0 全部通过后，再检查 P1。
5. P2 可以作为后续体验迭代项。
