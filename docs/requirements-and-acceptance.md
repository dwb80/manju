## 一、需求规格概述

### 1.1 项目名称

**Agnes AI Studio** —— AI漫剧工业化生产平台

### 1.2 一句话定位

对标 ChatGPT + Midjourney + Runway 的一站式 AI 创作平台，集成聊天、图片生成、视频生成三大核心能力，面向 AI 内容生产团队的企业级生产管理平台。

### 1.3 项目目标

1. 提供基于 Agnes AI 官方 API 的统一 AI 创作入口
2. 支持多模态（文本/图片/视频）输入输出与多轮上下文交互
3. 提供完整的会话管理、历史回溯、收藏能力
4. 打造轻量、可本地运行的个人 AI 创作工具
5. 支持从剧本创作到分镜、图片、视频、审核、发布的完整生产流程

### 1.4 项目范围

**范围内（In Scope）**：
- Web 端（PC 优先，响应式适配移动端）
- 会话系统、内容生成
- 三大核心能力：聊天（Agnes 2.0 Flash）、图片生成（Agnes Image 2.1 Flash）、视频生成（Agnes Video V2.0）
- 项目管理、剧本、分镜、角色、场景、资产、审核、发布

**范围外（Out of Scope）**：
- 移动原生 App（iOS/Android）
- 第三方开放平台 API 接入（V3 阶段考虑）
- 离线模型部署

### 1.5 术语约定

- **MUST**：强制需求，缺失则产品不可用
- **SHOULD**：建议需求，强烈推荐实现
- **MAY**：可选需求，根据资源情况决定

## 二、功能需求（按业务阶段）

### 2.1 项目立项阶段

**US-001: 创建新项目**
- 必填字段：项目名称（1-100字符）、季度信息（YYYY-QQ格式）、项目负责人
- 项目状态初始化为"规划中"，项目ID自动生成（PRJ-YYYYMMDD-序号）
- 项目负责人自动获得"管理员"权限
- 自动创建剧集管理模块、资产库、工作流配置
- **优先级**: P0

**US-002: 创建剧集**
- 剧集标题（1-100字符）、剧集序号（1-999）、关联项目
- 默认状态"待编写"，支持状态流转（待编写→编写中→待审核→已完成）
- 自动关联空白剧本对象、分镜管理模块、角色/场景资产库
- **优先级**: P0

### 2.2 剧本生产阶段

**US-003: 剧本版本管理**
- 支持从零创建新剧本，从现有版本创建新版本（自动复制内容）
- 版本号自动递增（v1.0→v1.1→v2.0）
- 支持版本对比（文本差异高亮显示）、版本回退
- 支持多人同时查看，编辑时自动锁定（30分钟超时）
- **优先级**: P0

**US-004: AI辅助剧本分析**
- 自动识别剧本中的人物，生成人物关系图谱
- 自动识别场景，提取场景要素（地点/时间/氛围/关键道具）
- 记录AI调用参数（模型/Token数/耗时）
- 失败时支持一键重试（最多3次，60秒超时）
- **优先级**: P1

### 2.3 分镜设计阶段

**US-005: 创建分镜**
- 镜头编号自动生成（SCENE-序号）、景别（远景/全景/中景/近景/特写）
- 角度（平视/俯视/仰视/侧视）、运镜方式（固定/推/拉/摇/移/跟）
- 时长（1-300秒）、画面描述（500字符）、对白内容（500字符）
- 支持AI自动生成图片Prompt、Prompt版本管理
- 支持拖拽调整分镜顺序、批量调整时长
- **优先级**: P0

**US-006: AI生成分镜建议**
- 根据剧本对白自动拆解镜头
- 智能推荐景别/角度/运镜方式
- 支持合并/拆分/删除/新增分镜，调整后自动重新编号
- **优先级**: P1

### 2.4 角色与场景资产阶段

**US-007: 角色资产管理**
- 角色名称（1-50字符）、角色定位（主角/配角/路人）
- 支持上传参考图（JPG/PNG，最大10MB）、AI生成角色图
- 三视图（正面/侧面/背面）、表情库（至少3种表情）
- Prompt管理（基础Prompt、扩展Prompt、模板、变量替换）
- LoRA管理（上传自定义LoRA，100MB限制）
- 分镜中可选择角色，引用时自动带入Prompt
- **优先级**: P0

**US-008: 场景资产管理**
- 场景名称（1-50字符）、场景类型（室内/室外/幻想）
- 时间设定（白天/黄昏/夜晚）、光照条件（自然光/人工光/特殊光）
- 支持上传参考图、AI生成场景图、多角度视图
- Prompt管理（基础Prompt、不同时间/光照的变体Prompt）
- **优先级**: P0

### 2.5 AI图片生产阶段

**US-009: AI图片生成**
- 从分镜选择生图任务，自动加载角色/场景Prompt
- 支持上传参考图（图生图模式）、选择AI模型
- 批量生成（队列管理：暂停/取消/重试）、显示生成进度
- 每次生成展示4张候选图，支持选择/下载/删除
- 记录生成参数（Prompt/模型/Seed/参考图）、Token消耗、耗时
- 显示单张图片成本估算、项目累计图片成本、预算预警
- **优先级**: P0

**US-010: 图片审核**
- 待审核图片列表，按项目/剧集/分镜筛选
- 通过/驳回/标记问题，驳回时填写原因
- 驳回图片自动标记为"待重制"，通知创作者
- 统计审核通过率、常见驳回原因
- **优先级**: P0

### 2.6 AI视频生产阶段

**US-011: AI视频生成**
- 从分镜选择生成视频任务，自动加载分镜图片
- 支持设置运动参数（镜头运动/角色动作）、时长（1-60秒）
- 批量提交任务、显示队列状态、支持取消排队中的任务
- 视频生成完成后自动关联分镜，支持在线预览、下载（MP4）
- 记录生成参数、GPU消耗、耗时、成本估算
- **优先级**: P0

**US-012: 视频审核**
- 待审核视频列表，支持在线播放
- 通过/驳回/标记问题，支持逐帧查看
- 问题分类：质量/风格/内容/技术
- **优先级**: P0

### 2.7 后期制作阶段

**US-013: 音频管理（P1）**
- 配音管理（上传/在线录音/AI配音）、音频剪辑
- 背景音乐（上传/音乐库选择/音量调整/循环播放）
- 音效管理（上传/音效库选择/时间轴添加）
- 音频同步（波形显示/对齐关键帧/音频预览）

**US-014: 字幕管理（P1）**
- 手动输入/导入SRT/VTT/AI语音识别/从剧本对白自动生成
- 时间轴编辑、批量调整时间、字幕样式设置
- 导出SRT/VTT格式、硬字幕/软字幕

**US-015: 视频剪辑（P1）**
- 时间轴编辑（拖拽排列/裁剪/调整顺序/删除）
- 转场效果（淡入淡出/溶解等）
- 导出MP4格式（720P/1080P/4K）

### 2.8 审核发布阶段

**US-016: 内容审核**
- 按项目/剧集筛选待审核内容
- 审核标准：内容合规性、版权审核、质量审核、剧情连贯性
- 通过/驳回/需修改，驳回时填写详细原因
- 支持打回指定环节（剧本/图片/视频）、多级审核（初审/复审）
- **优先级**: P0

**US-017: 发布管理（P1）**
- 选择发布平台、设置发布时间（立即/定时）、发布范围（公开/付费/会员）
- 发布预检（自动检查素材完整性、格式合规性）
- 一键发布、显示发布进度、发布失败时显示错误原因
- 发布记录（时间/平台/状态）、支持撤回

### 2.9 项目协作管理阶段

**US-018: 项目任务管理**
- 创建任务（标题、项目ID、状态、负责人、截止日期）
- 任务列表（按项目/状态/负责人筛选、按截止日期排序）
- 任务状态：todo/script/storyboard/image/video/review/done
- 任务统计（总数、已完成数、完成率）
- **优先级**: P0

**US-019: 项目问题管理**
- 创建问题（标题、项目ID、严重程度、状态、负责人、关联对象）
- 问题列表（按项目/状态/严重程度/负责人筛选）
- 问题状态流转：open → doing → resolved → closed
- 问题统计（总数、各严重程度数量、未解决数量、平均解决时间）
- **优先级**: P0

**US-020: 项目里程碑管理**
- 创建里程碑（标题、项目ID、状态、负责人、截止日期）
- 里程碑列表（按项目/状态筛选、按截止日期排序、显示进度条）
- 里程碑状态：planned/doing/done/delayed
- 里程碑统计（总数、已完成数、延期数、项目完成率）
- **优先级**: P0

**US-021: 剪辑片段管理**
- 创建剪辑片段（标题、项目ID、剧集、场景、镜头、关联分镜ID）
- 设置入点/出点时间、排序序号、状态
- 可视化时间轴（拖拽调整位置、裁剪时长、自动计算总时长）
- 导出剪辑清单（EDL/XML格式）、导出最终成片（MP4格式）
- **优先级**: P0

## 三、验收标准

### 3.1 验收标准格式

每个用户故事遵循以下格式：

```
### US-XXX: 功能名称
**用户故事**: 作为[角色],我想要[功能],以便[价值]。
**验收标准**:
- [ ] AC1: 验收条件名称
  - 具体验证点1
  - 具体验证点2
**优先级**: P0/P1/P2
**依赖**: US-XXX
**技术备注**: 技术实现要点
```

### 3.2 优先级定义

- **P0（MVP核心）**: 第一版本必须实现的核心功能
- **P1（重要功能）**: 重要但可延后实现的功能
- **P2（增强功能）**: 锦上添花的增强功能

### 3.3 验收标准编写原则

1. **可测试性**：每个AC必须有明确的验证方法
2. **完整性**：覆盖正常流程+异常流程+边界情况
3. **独立性**：每个AC独立可验证
4. **明确性**：避免模糊词汇，使用具体数值和条件

## 四、前端验收标准

### 4.1 优先级定义

- **P0**：上线阻断。白屏、核心功能不可用、数据串会话、保存失败、误删、导出不可用
- **P1**：重要体验问题。布局错位、空状态缺失、筛选不准、错误提示不清楚、移动端难用
- **P2**：体验优化。动效、微文案、视觉细节、非核心兼容性

### 4.2 通用验收

**视觉与布局（P0）**：
- 页面不能出现内容重叠、主要按钮不可见、输入框遮挡操作区
- 图片和视频必须按原比例展示，不允许拉伸变形
- 320px 到 1920px 宽度下核心页面可用，不出现破坏性横向滚动
- 文本不能撑破按钮、卡片、菜单和表格单元格

**交互（P0）**：
- 所有可点击按钮必须有有效结果，不能点击无反应
- 保存、删除、导出、生成、上传必须有成功或失败反馈
- 请求中的按钮要防重复提交
- 删除、移除、归档等危险操作必须二次确认

**功能逻辑（P0）**：
- 查询、提交、保存、删除、导出接口调用成功
- 接口失败或网络失败时页面不白屏，展示明确错误
- 会话、图片、视频、项目、资产必须绑定正确归属，不能串数据

**工程（P0）**：
- `npm run build` 必须通过
- 后端相关改动必须通过 `npm test`
- 不能留下导致构建失败的类型错误
- 不直接渲染用户输入的原生 HTML

### 4.3 专项验收

**聊天（P0）**：
- 新建会话后可以发送消息
- 一个会话正在生成时，新建其他会话不能中断原任务
- 停止生成、重新生成入口可用
- 点击历史会话后，消息展示区显示对应会话内容

**图片生成（P0）**：
- 支持多张图片附件上传和预览
- 生成结果绑定当前会话和当前项目
- 历史会话中的图片只显示在所属会话下
- 图片详情页能新标签页打开，支持查看、下载、复制、收藏、删除

**视频生成（P0）**：
- 视频生成任务绑定当前会话和项目
- 视频结果可播放、打开详情、下载
- 分镜底图资产可以直接带入图生视频
- Agnes 429 等限流错误要提示清楚，不能无限 loading

**项目工作台（P0）**：
- 项目首页、团队、剧集、问题、里程碑、剧本、分镜、剪辑、审核、任务、资产、交付工作页可正常切换
- 新增、更新、删除操作调用接口成功
- 项目不存在时要退出坏 ID，不能持续报 `project not found`

**资产库（P0）**：
- 支持图片、视频、角色、场景、风格、提示词、项目资料、分镜底图分类
- 资产可新增、删除、收藏、取消收藏
- 图片和视频资产可作为生成输入源复用
- 角色和场景资产可绑定到分镜

**交付导出（P0）**：
- 剧本文档 TXT、分镜表 CSV、剪辑清单 CSV、项目清单 JSON 可导出
- CSV 中文不乱码

### 4.4 验收命令

后端：
```bat
cd backend
npm test
```

前端：
```bat
cd frontend
npm run build
npm run test:e2e
```

## 五、API接口规范摘要

### 5.1 API设计原则

- RESTful 风格，资源命名使用复数名词
- URL层级不超过3级
- 使用HTTP标准方法（GET/POST/PUT/DELETE）
- 统一响应结构：`{ "code": 0, "message": "ok", "data": {} }`

### 5.2 认证与授权

- JWT Token认证，放置在 Authorization Header: `Bearer {token}`
- Token过期时间：24小时，支持刷新Token（7天）
- 所有API（除登录注册外）都需要验证Token

### 5.3 核心接口清单

| 模块 | Method | Path | 说明 |
|------|--------|------|------|

### 5.4 错误码定义

| 错误码 | HTTP状态码 | 说明 | 示例场景 |
|--------|-----------|------|---------|
| VALIDATION_ERROR | 400 | 数据验证失败 | 字段缺失/格式错误 |

| AUTHORIZATION_ERROR | 403 | 权限不足 | 无项目访问权限 |

### 5.5 API性能要求

| AI任务状态查询 | ≤200ms | 100并发 | 高频查询 |
| 文件上传API | ≤5秒 | 10并发 | 10MB文件 |

## 六、数据字典摘要

### 6.1 核心表结构

| 表名 | 用途 | 核心字段 |
|------|------|----------|
| `projects` | 项目基础信息 | id, project_code, name, status, owner_id, budget, ai_budget, ai_cost_used |
| `episodes` | 剧集规划 | id, episode_code, project_id, episode_number, title, status, script_id |
| `scripts` | 剧本版本 | id, script_code, episode_id, version, content, characters, scenes, is_current |
| `storyboards` | 分镜中心 | id, storyboard_code, episode_id, scene_number, shot_type, angle, camera_movement, duration, description, prompt, status |
| `characters` | 角色资产 | id, character_code, project_id, name, role_type, description, base_prompt, reference_images |
| `scenes` | 场景资产 | id, scene_code, project_id, name, scene_type, description, time_setting, lighting, base_prompt |
| `assets` | 通用资产 | id, asset_code, project_id, asset_type, name, file_url, file_size, review_status |
| `ai_tasks` | AI任务记录 | id, task_code, project_id, task_type, model_provider, model_name, status, progress, cost, duration |
| `users` | 用户 | id, username, email, password_hash, role, status |
| `project_members` | 项目成员 | id, project_id, user_id, role, permissions |
| `work_items` | 统一工作项 | id, project_id, title, status, owner, due_date（任务/问题/评审/里程碑） |
| `project_clips` | 剪辑片段 | id, project_id, storyboard_id, episode, scene, shot, title, duration, in_point, out_point, order_index, status |
| `loras` | 自定义模型 | id, lora_code, project_id, name, file_url, trigger_words |

### 6.2 通用字段约定

- 所有表必须有：`id`（主键）、`created_at`、`updated_at`
- 所有业务表支持软删除：`deleted_at` 字段
- 编码字段自动生成，格式：`类型-项目ID-序号`
- 时间字段统一使用 `_at` 结尾
- JSON 数据以 JSON 字符串形式存储

### 6.3 核心关系

| 关系 | 类型 | 说明 |
|------|------|------|
| Project → Episode | 1:N | 一个项目包含多个剧集 |
| Project → Character | 1:N | 一个项目包含多个角色 |
| Project → Scene | 1:N | 一个项目包含多个场景 |
| Episode → Script | 1:N | 一个剧集包含多个剧本版本 |
| Episode → Storyboard | 1:N | 一个剧集包含多个分镜 |
| Storyboard → Character | N:M | 一个分镜可关联多个角色 |
| Storyboard → Scene | N:1 | 一个分镜关联一个场景 |
| AITask → Asset | N:1 | 一个AI任务生成一个资产 |

### 6.4 数据库设计原则

- 主键使用 INTEGER AUTO_INCREMENT
- 外键必须设置引用关系和删除策略（ON DELETE CASCADE）
- 必填字段设置 NOT NULL
- 唯一字段设置 UNIQUE
- 使用 CHECK 约束验证数据范围
- 高频查询字段建立索引
- 复合查询建立复合索引
