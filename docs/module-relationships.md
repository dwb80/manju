## 一、核心概念确认

### 1.1 一个剧本的拆分关系

```
一个剧本
├── 可以拆分出多个角色
│   ├── 角色1：林逸
│   ├── 角色2：萧晓
│   ├── 角色3：茶信馆老板
│   └── ...
│
├── 可以拆分出多个场景
│   ├── 场景1：茶信馆
│   ├── 场景2：长街
│   ├── 场景3：客栈
│   └── ...
│
└── 可以拆分出多个分镜
    ├── 分镜1：镜头1-1（林逸推开木门）
    ├── 分镜2：镜头1-2（林逸走进茶信馆）
    ├── 分镜3：镜头1-3（萧晓抬头看林逸）
    └── ...
```

### 1.2 角色工厂和场景工厂定义（重要更正）

**角色工厂** = 生成角色图片的**生产车间**  
**场景工厂** = 生成场景图片的**生产车间**

它们不是资产库，而是**生产模块**！

```
AI生产中心
├── 图片生成（通用）
├── 视频生成（通用）
├── 角色工厂（专门生成角色图片）⭐ 生产车间
└── 场景工厂（专门生成场景图片）⭐ 生产车间

资产中心（仓库）
├── 角色资产（来自角色工厂的筛选结果）
└── 场景资产（来自场景工厂的筛选结果）
```

## 二、剧本中心与分镜导演台的关系

### 2.1 关系定义

| 概念 | 定义 | 职责 |
|------|------|------|
| **剧本中心** | 剧本创作阶段 | 创作、编辑、管理剧本内容（文字） |
| **分镜导演台** | 分镜设计阶段 | 将剧本拆分为分镜、设计镜头（视觉） |
| **关系** | 剧本提供内容，分镜进行拆分 | 剧本中心 → 分镜导演台 |

### 2.2 数据流转方向

```
剧本中心（输出）
├── 剧本内容（文字）
├── 角色设定（文字描述）
├── 场景设定（文字描述）
├── 对白内容
└── 剧情结构

    ↓ 提供内容

分镜导演台（输入）
├── 读取剧本内容
├── 拆分为多个分镜
├── 设计镜头参数
├── 分配角色和场景
└── 生成分镜图片
```

```

   ↓

   ↓

   ├── task_id: img-001

   ├── image_urls: ["角色1.png", "角色2.png", "角色3.png"]
   └── status: completed
   ↓
4. 用户查看生成的图片
   ↓
5. 用户评估图片质量
   ├── 角色特征是否准确？
   ├── 一致性是否良好？
   └── 是否符合剧本设定？
   ↓
6. 用户选择可用的图片
   ├── 标记为"可用"
   └── 或重新生成
   ↓
7. 保存到资产中心
   ├── 创建角色资产
   ├── 填写角色名称
   ├── 保存角色描述（prompt）
   ├── 保存选中的图片URL
   └── 绑定到项目
```

## 四、剧本中心与模型中心的关系

### 4.1 集成目标

剧本中心需要使用模型中心的AI模型来支持以下功能：
- AI剧本创作（聊天模型）
- AI剧本优化（聊天模型）
- AI生成场景描述（聊天模型）
- AI生成对白（聊天模型）
- AI拆分镜头（聊天模型）
- AI连续性检查（聊天模型）

### 4.2 目标架构

```
剧本中心 → 模型中心服务 → AI能力层 → AgnesClient → Agnes API
```

### 4.4 剧本AI服务层接口

| 功能 | 方法 | 说明 |
|------|------|------|
| AI剧本创作 | `createScript(params)` | 基于提示词生成剧本内容 |
| AI剧本优化 | `optimizeScript(params)` | 对现有剧本进行优化（扩写/压缩/改写/润色） |
| AI生成场景描述 | `generateScene(params)` | 基于剧本内容生成场景描述 |
| AI生成对白 | `generateDialogue(params)` | 基于角色和场景生成对白 |
| AI拆分镜头 | `breakdownScript(params)` | 将剧本拆分为分镜列表 |
| AI连续性检查 | `checkContinuity(params)` | 检查剧本中的角色/时间/地点一致性 |

## 六、分镜拆分流程

### 6.1 手动拆分流程

```
1. 用户在剧本中心完成剧本创作
   ├── Scene01: 茶信馆
   │   ├── 林逸：走了出来
   │   └── 萧晓：终于舍得出来了
   └── Scene02: 长街
       └── 林逸：与你无关

    ↓

2. 用户进入分镜导演台
   ├── 选择剧本（从剧本中心）
   └── 查看剧本内容

    ↓

3. 用户手动拆分分镜
   ├── Scene01拆分为3个镜头
   │   ├── 镜头1-1：林逸推开木门（中景）
   │   ├── 镜头1-2：林逸走进茶信馆（跟镜头）
   │   └── 镜头1-3：萧晓抬头看林逸（特写）
   └── Scene02拆分为2个镜头
       ├── 镜头2-1：林逸走在长街上（远景）
       └── 镜头2-2：林逸停下脚步（中景）

    ↓

4. 用户填写分镜参数
   ├── 角色：引用角色资产ID
   ├── 场景：引用场景资产ID
   ├── 景别：远景/中景/近景/特写
   ├── 运动：推/拉/摇/移/跟
   └── 时长：3秒/5秒/10秒

    ↓

5. 系统自动生成提示词
   ├── 基于角色资产：角色外观、服装、特征
   ├── 基于场景资产：场景氛围、光影、色调
   └── 基于分镜描述：动作、对白、镜头运动

    ↓

6. AI生成分镜图片
   ├── 调用图片生成API
   └── 保存图片URL
```

### 6.2 AI自动拆分流程

```
1. 用户在剧本中心完成剧本创作

    ↓

2. 用户点击【AI拆分分镜】按钮

    ↓

3. AI读取剧本内容
   ├── 解析剧本结构
   ├── 提取场景、角色、对白
   └── 理解剧情节奏

    ↓

4. AI自动生成分镜列表
   ├── 每个场景拆分为多个镜头
   ├── 推荐景别、角度、运动方式
   └── 生成镜头描述和Prompt

    ↓

5. 用户审核和调整
   ├── 查看AI生成的分镜列表
   ├── 调整镜头顺序
   ├── 修改镜头参数
   └── 确认或重新生成

    ↓

6. 保存分镜数据
   ├── 保存到 project_storyboards 表
   ├── 关联角色资产ID
   ├── 关联场景资产ID
   └── 生成分镜图片任务
```

## 七、数据库关联设计

### 7.1 project_storyboards 表（分镜表）

```sql
CREATE TABLE project_storyboards (
    id                  TEXT PRIMARY KEY,
    project_id          TEXT NOT NULL,              -- 项目绑定 ⭐
    
    -- 剧本关联字段
    episode             INTEGER,                    -- 剧集号
    scene               TEXT,                       -- 场景名称
    shot                TEXT,                       -- 镜头编号
    
    -- 内容字段
    title               TEXT,                       -- 分镜标题
    description         TEXT,                       -- 分镜描述
    dialogue            TEXT,                       -- 对白内容
    
    -- 资产引用字段 ⭐ 关键
    characters          TEXT,                       -- 角色名称列表（JSON）
    character_asset_ids TEXT,                       -- 角色资产ID列表（JSON）⭐
    location            TEXT,                       -- 地点名称
    scene_asset_id      TEXT,                       -- 场景资产ID ⭐
    
    -- 镜头参数
    shot_size           TEXT,                       -- 景别
    camera_move         TEXT,                       -- 镜头运动
    duration            INTEGER,                    -- 时长（秒）
    
    -- AI生成字段
    prompt              TEXT,                       -- 提示词
    image_task_id       TEXT,                       -- 图片任务ID
    image_url           TEXT,                       -- 图片URL
    video_task_id       TEXT,                       -- 视频任务ID
    video_url           TEXT,                       -- 视频URL
    
    -- 状态字段
    status              TEXT DEFAULT 'draft',       -- 状态
    notes               TEXT,                       -- 备注
    
    -- 时间字段
    created_at          TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at          TEXT DEFAULT CURRENT_TIMESTAMP,
    
    -- 索引
    INDEX idx_project_episode (project_id, episode),
    INDEX idx_scene_asset (scene_asset_id),
    INDEX idx_status (status)
);
```

### 7.2 关联字段说明

| 字段 | 关联对象 | 说明 |
|------|---------|------|
| **character_asset_ids** | 角色资产 | 引用角色资产ID列表（如 ["role-12", "role-15"]） |
| **scene_asset_id** | 场景资产 | 引用场景资产ID（如 "scene-25"） |
| **project_id** | 项目 | 剧本和分镜必须绑定同一项目 |
