# 产品正式立项

## 项目名称（暂定）

> **AI漫剧项目管理系统（AIGC Drama Production Platform）**

后续如果扩展，我们会升级为：

> **AI内容生产中台（AI Content Production Platform，AICP）**

这样 V1 做漫剧，未来可以无缝扩展：

```
AI内容生产中台

├── AI漫剧
├── AI短剧
├── AI漫画
├── AI小说
├── AI广告
├── AI宣传片
├── AI课程
├── AI直播
└── AI数字人
```

所以我们的数据库和架构都会按**中台**设计，而不是只做漫剧。

# 第一章：Vision（产品愿景）

## 一、产品定位

> 一套面向 AI 内容生产团队的企业级生产管理平台。

它不是：

❌ AI 绘图工具

不是：

❌ AI 视频工具

不是：

❌ Jira

也不是：

❌ 禅道

而是：

> **AI生产流水线管理平台**

---

## 二、产品使命（Mission）

一句话描述：

> **让 AI 内容生产从"个人创作"升级为"工业化流水线生产"。**

例如：

以前：

```
编剧
↓

美术

↓

导演

↓

剪辑
```

每个人都在不同软件工作。

以后：

```
一个平台

↓

统一资产

↓

统一模型

↓

统一工作流

↓

统一审核

↓

统一发布
```

---

## 三、产品价值

系统帮助企业解决以下问题：

| 问题          | 系统解决方案              |
| ----------- | ------------------- |
| 剧本版本混乱      | 版本管理、变更记录、多人协作      |
| Prompt 到处复制 | Prompt 模板库、版本控制、变量化 |
| 角色风格不统一     | 角色资产库 + 一致性引用       |
| 场景重复制作      | 场景资产库复用             |
| 图片无法追溯      | AI 任务记录、生成参数留档      |
| 视频生成失败      | 自动重试、队列管理、失败诊断      |
| AI 成本不可控    | Token/GPU/API 成本统计  |
| 流程混乱        | 可配置工作流与节点状态管理       |
| 成果难复用       | 项目级与企业级资产沉淀         |

## 五、技术定位

采用开放式 AI Provider 架构：

```
                AI生产平台

                       │

         ┌─────────────┼─────────────┐

     LLM Provider   Image Provider   Video Provider

         │                │                │
```

## 六、产品设计原则

这是后续所有功能设计都要遵循的原则。

### 原则一：资产优先（Asset First）

任何可复用内容都应沉淀为资产，例如：

* 角色
* 场景
* Prompt
* 音效
* 配音
* LoRA
* 工作流
* 模板

目标是避免重复创建。

---

### 原则二：AI 原生（AI Native）

AI 不只是一个按钮，而是贯穿整个生产流程。

例如：

```
剧本
    ↓ AI 优化
分镜
    ↓ AI 生成
Prompt
    ↓ AI 优化
图片
    ↓ AI 批量生成
视频
    ↓ AI 生成
审核
    ↓ AI 预审
发布
    ↓ AI 推荐发布时间
```

---

### 原则三：工作流驱动（Workflow Driven）

不是页面驱动，而是流程驱动。

例如：

```
待编剧

↓

待分镜

↓

待图片

↓

待视频

↓

待审核

↓

已发布
```

系统围绕状态推进，而不是让用户自己管理进度。

---

### 原则四：全链路可追溯（Traceability）

任何内容都应能追溯到来源。

例如：

```
视频

↓

由哪些图片生成

↓

图片使用哪个 Prompt

↓

Prompt 使用哪个模板

↓

模板属于哪个项目

↓

由哪个成员修改
```

形成完整的血缘关系（Lineage）。

---

### 原则五：平台化（Platform First）

所有 AI 能力都通过统一接口调用。

例如：

```
AI图片

↓

统一 AI Service

↓

Provider

↓

即梦
Flux
SDXL
ComfyUI
```

业务模块无需关心具体模型。
