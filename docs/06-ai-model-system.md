本文档定义了 AI 漫剧项目管理系统的AI模型体系设计，包括模型分类、用户选择模式、模型策略中心等核心设计。这是AI能力架构的核心组成部分。

# 产品设计调整：AI模型体系

## 核心问题

> **AI漫剧生产过程中，不同环节需要不同类型的模型，不能让一个"大模型"包打天下。**

设计方向需要从"模型调用"升级为：

> **场景驱动的模型编排体系（Model Orchestration）**

---

## 架构升级

### 原方案

```
业务模块
    |
    |
AI Provider
    |
    |
各种模型
```

### 升级为

```
业务场景
    |
    |
AI能力层
    |
    |
模型路由层
    |
    |
具体模型
```

---

# 一、模型不按厂商分类，而按能力分类

系统不能简单展示：

```
模型A
模型B
模型C
```

用户不知道怎么选。

应该设计：

## AI能力分类

### 1. 文本推理模型（编剧模型）

#### 用途

* 剧情设计
* 人物关系分析
* 世界观构建
* 剧本扩写
* 台词优化
* 分镜拆解
* 连贯性检查

#### 特点

重点：

* 推理能力
* 长上下文
* 文学理解
* 逻辑能力

例如：

```
剧本创作 Agent

↓

调用文本推理模型
```

---

### 2. 文生图模型

#### 用途

* 角色设计
* 场景设计
* 分镜图
* 海报

#### 重点

* 画面质量
* 风格一致性
* 参考图理解

---

### 3. 图生图模型

#### 用途

非常重要。

例如：

已有角色：

```
林逸正面图

↓

生成：

侧面

背面

战斗姿态

不同表情
```

#### 重点

* 保持角色一致性

---

### 4. 视频生成模型

#### 用途

* 图片转视频
* 文生视频
* 动作生成
* 镜头运动

---

### 5. 音频模型

#### 包括

* TTS配音
* 音效生成
* BGM生成

---

# 二、用户选择模型方式调整

不应该让普通用户直接选择模型。

建议设计三级模式。

---

## 模式1：智能模式（普通用户）

用户只选择：

例如：

```
任务：

生成剧本

生成角色

生成视频

```

系统自动选择：

```
任务类型
+
质量要求
+
成本限制
+
速度要求

↓

自动匹配模型
```

---

## 模式2：专家模式（专业用户）

用户可以选择：

### 剧本生成

```
模型：

[下拉选择]

上下文长度

推理等级

温度

```

---

### 图片生成

```
模型：

[选择]

画质：

普通
高清
电影级

速度：

快速
质量优先
```

---

### 视频生成

```
模型：

[选择]

时长

分辨率

运动强度

一致性等级
```

---

## 模式3：企业管理员模式

管理整个企业模型池。

例如：

```
企业模型中心

--------------------------------

文本模型

模型1
状态：启用
费用：
适用场景：

--------------------------------

图片模型

模型2

--------------------------------

视频模型

模型3

```

管理员可以：

* 添加模型
* 禁用模型
* 设置权限
* 设置默认模型
* 设置费用

# 调整后的模型选择体系

## 核心原则

系统默认：

> **任务优先，而不是模型优先。**

用户创建任务时：

例如：

```
生成角色三视图
```

系统默认：

```
角色设计任务模板

↓

自动选择推荐图片模型

↓

生成结果
```

用户无需关心：

* Flux
* SDXL
* 即梦
* Midjourney
* ComfyUI

---

但是：

专业用户可以切换：

```
当前模型：
默认推荐模型

[切换模型]
```

## AI模型选择策略

### 方案A：完全开放模型市场型

```
用户可以接入任何模型
类似插件生态
```

优点：扩展性最高
缺点：管理复杂

---

### 方案B：企业模型池型（推荐）

```
管理员维护可用模型

普通用户从模型池选择
```

优点：

* 企业可控
* 成本可控
* 适合私有化部署

---

### 方案C：智能路由型

```
用户只选择目标：

高质量
快速
低成本

系统自动选模型
```

优点：

* 使用简单
* AI体验最好

---

### 推荐方案

**企业版采用 B+C 混合：**

> 企业维护模型池，普通用户使用智能路由，专业用户可手动选择模型。

这个方案最符合 AI 漫剧生产团队的实际使用场景。

---

# AI漫剧生产核心驱动方式

A. **项目驱动型（传统影视公司模式）**

> 一个项目 → 多集 → 分镜 → 任务

B. **内容资产驱动型（AI时代模式）**

> 角色/场景/Prompt资产先沉淀，再组合生产

C. **工作流驱动型（自动化工厂模式）**

> 配置生产流水线，AI自动执行

### 建议方案

> **A+B+C融合：项目负责管理，资产负责复用，工作流负责自动生产。**
