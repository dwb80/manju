## 一、产品愿景与定位

### 1.1 产品定位

> 一套面向 AI 内容生产团队的企业级生产管理平台。

它不是：AI 绘图工具、AI 视频工具、Jira、禅道。

而是：**AI生产流水线管理平台**

### 1.2 产品使命

> **让 AI 内容生产从"个人创作"升级为"工业化流水线生产"。**

以前：编剧 → 美术 → 导演 → 剪辑（每个人都在不同软件工作）

以后：一个平台 → 统一资产 → 统一模型 → 统一工作流 → 统一审核 → 统一发布

### 1.3 产品价值

| 问题 | 系统解决方案 |
|------|-------------|
| 剧本版本混乱 | 版本管理、变更记录、多人协作 |
| Prompt 到处复制 | Prompt 模板库、版本控制、变量化 |
| 角色风格不统一 | 角色资产库 + 一致性引用 |
| 场景重复制作 | 场景资产库复用 |
| 图片无法追溯 | AI 任务记录、生成参数留档 |
| 视频生成失败 | 自动重试、队列管理、失败诊断 |
| AI 成本不可控 | Token/GPU/API 成本统计 |
| 流程混乱 | 可配置工作流与节点状态管理 |
| 成果难复用 | 项目级与企业级资产沉淀 |

### 1.4 产品边界

**V1 范围（必须完成）**: 项目管理、剧本、分镜、角色、场景、素材、AI图片、AI视频、Prompt、审核、发布、数据统计

**V2 范围**: Agent 自动创作、Agent 自动审核、Agent 自动发布、Agent 协同、MCP Marketplace

**V3 范围**: AI小说、AI漫画、AI动画、数字人、直播、企业知识库

### 1.5 技术定位

采用开放式 AI Provider 架构：

```
                AI生产平台
                       │
         ┌─────────────┼─────────────┐
     LLM Provider   Image Provider   Video Provider
         │                │                │
  GPT Claude Gemini   Flux SDXL 即梦   可灵 Runway Pika
```

任何模型都可以通过适配器接入，不绑定具体厂商。

### 1.6 产品设计原则

**原则一：资产优先（Asset First）**
任何可复用内容都应沉淀为资产（角色、场景、Prompt、音效、配音、LoRA、工作流、模板）。

**原则二：AI 原生（AI Native）**
AI 不只是一个按钮，而是贯穿整个生产流程（剧本→AI优化→分镜→AI生成→Prompt→AI优化→图片→AI批量生成→视频→AI生成→审核→AI预审→发布→AI推荐）。

**原则三：工作流驱动（Workflow Driven）**
不是页面驱动，而是流程驱动。系统围绕状态推进（待编剧→待分镜→待图片→待视频→待审核→已发布）。

**原则四：全链路可追溯（Traceability）**
任何内容都应能追溯到来源，形成完整的血缘关系（Lineage）。

**原则五：平台化（Platform First）**
所有 AI 能力都通过统一接口调用，业务模块无需关心具体模型。

## 二、用户角色与权限

### 2.1 角色一览

| 角色 | 核心职责 | 系统权限 |
|------|----------|----------|
| 项目管理员 | 管理项目、成员、流程 | 全项目 |
| 制片人 | 项目规划、成本、进度 | 项目管理 |
| 编剧 | 剧本创作、AI文本生成 | 剧本 |
| 分镜导演 | 镜头设计、分镜调整 | 分镜 |
| 美术设计师 | 角色、场景、画面设计 | 资产 |
| 视频导演 | 视频生成、镜头调整 | 视频 |
| 配音人员 | 声音制作 | 音频 |
| 剪辑人员 | 成片制作 | 成片 |
| 审核人员 | 内容审核 | 审核 |
| 发布运营 | 发布和数据分析 | 发布 |
| AI管理员 | 模型、Prompt、成本管理 | AI中心 |

### 2.2 权限设计原则

不是简单：用户 → 菜单权限

而是：用户 → 角色 → 项目权限 → 资源权限 → 操作权限

**核心设计要点**：
1. **项目级权限隔离**：用户在不同项目中可能有不同的权限级别
2. **资源级操作权限**：除了查看权限，还需要定义创建、编辑、删除等操作权限
3. **角色继承与组合**：允许一个用户同时拥有多个角色，权限自动叠加
4. **权限动态调整**：支持项目负责人临时调整成员权限

### 2.3 权限管理最佳实践

- **最小权限原则**：只授予用户完成工作所需的最小权限
- **权限分离原则**：关键操作需要多个角色共同确认
- **权限审计原则**：定期审计权限分配，及时清理不必要的权限
- **权限继承原则**：合理设计角色层级，利用继承机制简化权限管理

## 三、业务流程设计

整个生产流程包括 **8 个核心阶段**，从项目立项到最终发布：

```
项目立项 → 剧本生产 → 分镜设计 → 角色与场景资产 → AI图片生产 → AI视频生产 → 后期制作 → 审核发布
```

每个阶段都有明确的：输入、输出、产生对象、关键活动、AI辅助、人工把控。

### 3.1 阶段1：项目立项

- **输入**: 小说/IP/原创想法
- **输出**: 漫剧项目
- **产生对象**: 项目、剧集规划、团队成员
- **关键活动**: 项目创建、团队组建、剧集规划

### 3.2 阶段2：剧本生产

- **输入**: 原始剧情
- **AI辅助**: 文本推理模型 → 剧情分析 → 人物关系 → 剧本改编 → 对白优化
- **输出**: 剧本版本
- **产生对象**: Script
- **关键活动**: 剧情分析、剧本改编、剧本优化、剧本审核

### 3.3 阶段3：分镜设计

- **输入**: 剧本
- **AI分析**: 文本模型 → 拆解镜头 → 生成分镜建议
- **人工调整**: 导演修改 → 确定镜头语言
- **输出**: Storyboard
- **分镜包含**: 镜头编号、景别、角度、运动、人物、动作、对白、Prompt、参考资产
- **关键活动**: 镜头拆解、分镜脚本制作、导演调整、分镜审核

### 3.4 阶段4：角色与场景资产

**角色资产制作流程**: 人物设定 → 角色资产 → 三视图 → 表情库 → 动作库 → 可复用角色

**角色资产包含**: 基础设定、参考图、Prompt、LoRA、三视图、表情库、动作库

**场景资产制作流程**: 地点设定 → 场景资产 → 时间 → 天气 → 光照 → 镜头适配

**场景资产包含**: 建筑设定、环境设定、时间设定、光照设定、镜头适配

### 3.5 阶段5：AI图片生产

**流程**: 分镜 → 调用图片模型 → 生成图片 → 审核 → 入库资产

**记录**: Prompt、模型、参数、Seed、参考图、生成历史

**关键活动**: Prompt优化、图片生成、图片审核、图片入库

### 3.6 阶段6：AI视频生产

**输入**: 分镜图片
**执行**: 视频模型 → 图生视频 → 动作优化 → 镜头运动
**输出**: 视频片段

**关键活动**: 视频生成策略、视频生成执行、视频质量把控、视频入库

### 3.7 阶段7：后期制作

包括：配音、字幕、BGM、音效、剪辑

### 3.8 阶段8：审核发布

**审核**: 内容审核、画面审核、版权审核、质量审核
**通过**: 发布平台

## 四、核心业务对象

整个系统围绕以下核心对象构建：

### 4.1 Project（项目）

代表一部漫剧。包含：项目名称、季度、负责人、状态、成员。

### 4.2 Episode（剧集）

代表一集内容。关系：Project 1:N Episode。

### 4.3 Script（剧本）

文字内容。包含：正文、版本、AI分析结果、人物、场景。

### 4.4 Storyboard（分镜）

最核心对象。关系：Episode 1:N Storyboard。

一个分镜：镜头编号 → 图片 → 视频 → 音频

### 4.5 Character（角色资产）

包含：基础设定、参考图、Prompt、LoRA、表情、动作。

### 4.6 Scene（场景资产）

包含：建筑、环境、时间、光照。

### 4.7 AITask（AI任务）

所有AI生成统一抽象。包含：模型、参数、输入、输出、耗时、费用。

### 4.8 Workflow（工作流）

控制生产流程。

### 4.9 核心关系图

```
                 Project
                    |
              +-----+------+
              |            |
          Episode       Character
              |
          Storyboard
              |
       +------+------+
       |             |
    Image        Video
       |
    Asset

Script → Storyboard
Workflow → AITask → Model
```

### 4.10 数据模型设计原则

1. **领域驱动设计（DDD）**：以业务领域为核心
2. **资产优先原则**：所有可复用内容都设计为资产对象
3. **可追溯性原则**：建立完整的对象关系链
4. **版本管理原则**：关键对象支持版本管理
5. **状态管理原则**：所有生产对象都有明确的状态定义

## 五、AI模型体系

### 5.1 核心问题

> AI漫剧生产过程中，不同环节需要不同类型的模型，不能让一个"大模型"包打天下。

设计方向从"模型调用"升级为：**场景驱动的模型编排体系（Model Orchestration）**

### 5.2 架构升级

**原方案**: 业务模块 → AI Provider → 各种模型

**升级为**: 业务场景 → AI能力层 → 模型路由层 → 具体模型

### 5.3 AI能力分类

| 能力类型 | 用途 | 重点 |
|----------|------|------|
| 文本推理模型 | 剧情设计、人物关系、剧本扩写、台词优化、分镜拆解 | 推理能力、长上下文、文学理解 |
| 文生图模型 | 角色设计、场景设计、分镜图、海报 | 画面质量、风格一致性 |
| 图生图模型 | 角色一致性（侧面/背面/表情/动作） | 保持角色一致性 |
| 视频生成模型 | 图片转视频、动作生成、镜头运动 | 动作、人物一致、镜头运动 |
| 音频模型 | TTS配音、音效生成、BGM生成 | 声音质量、风格统一 |

### 5.4 用户选择模型方式（三级模式）

**模式1：智能模式（普通用户）**
用户只选择任务类型（生成剧本/生成角色/生成视频），系统自动匹配模型。

**模式2：专家模式（专业用户）**
用户可以选择具体模型和参数（模型、画质、速度、时长、分辨率等）。

**模式3：企业管理员模式**
管理整个企业模型池（添加、禁用、设置权限、设置费用）。

### 5.5 模型策略中心

企业级系统必须有的模块：

```
系统设置
  └── AI模型管理
       ├── 模型列表
       ├── 能力标签
       ├── 路由策略
       ├── 成本策略
       ├── 权限控制
       └── 调用统计
```

### 5.6 模型能力标签

每个模型不是简单名称，而是能力描述：

```
模型：xxx
能力：[文本推理][长文本][角色分析]
评分：推理能力 ★★★★★  速度 ★★★★  成本 ★★★
适合：剧本创作、剧情分析
```

### 5.7 推荐方案

**企业版采用 B+C 混合**：
> 企业维护模型池，普通用户使用智能路由，专业用户可手动选择模型。

### 5.8 AI任务记录

每一次生成都必须保存：任务ID、项目、分镜、使用模型、Prompt、参考图、参数、生成时间、耗时、消耗、结果、评分。

## 六、信息架构设计

### 6.1 首页工作台设计原则

采用：**A 项目驾驶舱 + B AI创作工作台 + C AI生产流水线监控 三合一**

不同角色进入系统，关注的信息完全不同：
- 项目负责人关心：项目是否按计划推进
- 创作者关心：我今天要做什么、AI怎么帮我完成
- 管理员关心：模型、成本、任务队列是否正常

### 6.2 顶层导航设计

```
AI漫剧生产平台
├── 首页工作台
├── 项目中心
├── 创作中心
├── AI生产中心
├── 资产中心
├── 审核中心
├── 发布中心
├── 数据中心
└── 系统管理
```

### 6.3 一级模块设计

| 模块 | 定位 | 核心功能 |
|------|------|----------|
| **首页工作台** | 用户每天进入系统的第一入口 | 项目驾驶舱、AI创作工作台、AI生产流水线监控 |
| **项目中心** | 整个系统业务主入口 | 我的项目、项目列表、项目创建、项目成员、项目计划、项目任务、项目归档 |
| **创作中心** | AI时代区别于传统项目管理的地方 | 剧本创作、分镜设计、Prompt工作台、AI助手、创作历史 |
| **AI生产中心** | 模型调用中心 | 图片生成、视频生成、文本生成、音频生成、AI任务队列、模型中心、调用记录 |
| **资产中心** | 核心竞争力 | 角色资产、场景资产、道具资产、图片资产、视频资产、音频资产、Prompt资产、工作流资产 |
| **审核中心** | 质量把控 | 待审核、我的审核、审核记录、质量评分、问题库 |
| **发布中心** | 内容交付 | 成片管理、发布计划、平台管理、数据回收、内容复盘 |
| **数据中心** | 决策支持 | 项目数据、AI数据、内容数据 |
| **系统管理** | 系统配置 | 用户管理、权限管理、企业设置、模型管理、Prompt模板、工作流管理、API配置、日志管理 |

### 6.4 项目详情页 Tab 结构

```
灵契界
--------------------------------
概览 | 剧本 | 分镜 | 角色 | 场景 | 素材 | AI任务 | 审核 | 发布 | 数据 | 设置
```

### 6.5 信息架构设计原则

1. **用户角色驱动**：根据不同用户角色设计不同的信息展示和操作入口
2. **任务流程驱动**：围绕生产流程组织信息结构
3. **资产优先原则**：资产中心作为核心竞争力，提供独立的资产管理和复用入口
4. **AI能力集成**：创作中心和AI生产中心体现AI时代的产品特点
5. **数据驱动决策**：数据中心提供全面的数据分析和决策支持能力
6. **可扩展性设计**：支持后续功能扩展（AI小说、AI漫画、数字人等）

## 七、非功能性需求

### 7.1 性能需求

| 操作类型 | 响应时间要求 | 备注 |
|----------|-------------|------|
| 首页加载(初始) | ≤ 2秒 | 首屏渲染完成 |
| 项目列表页加载 | ≤ 1秒 | 含20个项目 |
| 按钮点击响应 | ≤ 200ms | 立即反馈 |
| 数据查询API | ≤ 500ms | 单表查询 |
| 文本生成(剧本分析) | ≤ 30秒 | 或60秒超时 |
| 图片生成(单张) | ≤ 30秒 | 4张候选图 |
| 视频生成(单个) | ≤ 5分钟 | 或异步处理 |

**并发处理能力**：
- 正常负载：50用户，响应时间正常
- 高峰负载：200用户，响应时间可增加50%
- 极端负载：500用户，系统稳定，允许降级服务

**数据容量（V1）**：
- 项目数量：100个
- 剧集数量：1000集
- 分镜数量：10,000个
- 图片资产：50,000张
- 视频资产：5,000个
- 用户数量：500用户

### 7.2 安全需求

**身份认证**：
- JWT Token认证，过期24小时，支持刷新Token（7天）
- 密码加密存储（bcrypt/加盐）
- 基于角色的访问控制（RBAC）
- 项目级权限隔离

**数据安全**：
- 所有API使用HTTPS（TLS 1.2+）
- 数据库密码加密存储
- 敏感字段加密
- 定期数据备份（每日备份）

**防护机制**：
- SQL注入防护（参数化查询）
- XSS攻击防护（输入过滤+输出编码）
- CSRF防护（Token验证）
- API滥用防护（请求频率限制：100次/分钟）
- AI API密钥加密存储

### 7.3 可用性需求

| 指标 | 目标值 |
|------|--------|
| 系统可用率 | 99.5%（每月停机时间≤3.6小时） |
| 故障恢复时间(MTTR) | ≤ 30分钟 |
| 数据恢复时间 | ≤ 4小时 |

**容错机制**：
- 数据库主从备份
- 服务异常自动重启
- AI API调用失败重试（最多3次）
- AI服务降级策略（默认模型不可用时切换备用模型）

**备份策略**：
- 数据库全量备份：每日
- 数据库增量备份：每小时
- 文件资产备份：每周
- 配置文件备份：每次变更

### 7.4 可扩展性需求

**架构可扩展性**：
- 支持多实例部署（无状态设计）
- 支持负载均衡（Nginx/K8s）
- 支持数据库分库分表（V2阶段）
- 支持文件存储扩展（对象存储）

**AI服务可扩展性**：
- 支持新增AI模型（无需修改代码）
- 支持模型配置管理
- 支持模型性能对比
- 支持模型版本管理

**业务扩展能力（V2/V3）**：
- AI短剧、AI漫画、AI小说、AI广告、AI课程、数字人直播

### 7.5 易用性需求

**设计原则**：
- 一致的视觉风格（配色/字体/图标）
- 清晰的导航结构（最多3级）
- 响应式设计（支持桌面端/平板）
- 即时反馈（操作成功/失败提示）
- 加载状态显示（进度条/骨架屏）
- 错误信息友好（具体原因+解决建议）

**新用户引导**：
- 系统使用手册（在线文档）
- 新手引导流程（首次登录引导）
- 关键功能提示（Tooltip/帮助按钮）
- 示例项目（演示项目供学习）

### 7.6 兼容性需求

| 浏览器 | 版本要求 | 支持度 |
|--------|---------|--------|
| Chrome | 最新2个版本 | 100% |
| Safari | 最新2个版本 | 100% |
| Firefox | 最新2个版本 | 100% |
| Edge | 最新2个版本 | 100% |

**设备兼容性**：
- 桌面显示器：≥1280x720（最佳体验）
- 笔记本电脑：≥1366x768（正常体验）
- 平板电脑：≥1024x768（基础功能可用）
- 手机：≥375x667（仅查看功能）

### 7.7 稳定性需求

- 系统崩溃次数：≤ 1次/月
- 数据丢失次数：0次
- 内存泄漏：无（定期检测）
- CPU占用率：≤ 80%（正常负载）
- 错误率：≤ 0.5%（API请求）

### 7.8 可维护性需求

**代码质量**：
- 统一的编码规范（ESLint/Prettier）
- 代码注释率：≥ 30%（关键逻辑）
- 函数复杂度：≤ 10（单个函数）
- 代码重复率：≤ 5%

**测试覆盖**：
- 单元测试覆盖率：≥ 70%
- 集成测试覆盖核心流程
- E2E测试覆盖关键用户场景

### 7.9 监控与运维需求

**监控指标**：
- 系统监控：服务器CPU/内存/磁盘使用率、数据库连接数/查询性能、API响应时间/错误率
- 业务监控：项目创建数/完成数、AI生成任务数/成功率、AI成本消耗统计、用户活跃度统计

**告警机制**：
- 系统异常告警（邮件+短信）
- 性能下降告警（响应时间>阈值）
- AI成本超预算告警
- 安全事件告警

**日志管理**：
- 日志格式统一（JSON格式）
- 日志级别分明（ERROR/WARN/INFO/DEBUG）
- 日志保留时长：30天（系统日志）/ 90天（审计日志）
