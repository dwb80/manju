# 验收标准文档（Acceptance Criteria）

> 本文档为《AI漫剧项目管理系统》V1.0核心功能模块的验收标准,按照业务流程阶段进行组织。

---

## 1. 项目立项阶段（Project Initialization）

### US-001: 创建新项目

**用户故事**:
作为**项目负责人**,我想要**创建新的漫剧项目**,以便**启动新的漫剧生产流程**。

**验收标准**:

- [ ] AC1: 必填字段验证
  - 项目名称(必填,长度1-100字符)
  - 季度信息(必填,格式:YYYY-QQ)
  - 项目负责人(必填,从系统用户列表选择)
  - 项目描述(选填,最大1000字符)

- [ ] AC2: 项目状态初始化
  - 默认状态为"规划中"
  - 创建时间为系统当前时间
  - 项目ID自动生成(格式:PRJ-YYYYMMDD-序号)

- [ ] AC3: 成员权限分配
  - 项目负责人自动获得"管理员"权限
  - 支持添加其他成员并分配角色(导演/编剧/美术/审核员)
  - 权限立即生效

- [ ] AC4: 项目工作区初始化
  - 自动创建剧集管理模块
  - 自动创建资产库(角色库/场景库)
  - 自动创建工作流配置

- [ ] AC5: 数据持久化
  - 所有数据保存到数据库
  - 创建失败时回滚所有操作
  - 保存成功后显示项目详情页

**优先级**: P0(MVP核心)

**依赖**: 用户管理系统

**技术备注**:
- 数据库事务保证原子性
- 项目ID生成算法需保证唯一性
- 需要权限系统支持

---

### US-002: 创建剧集

**用户故事**:
作为**导演**,我想要**在项目下创建新剧集**,以便**规划漫剧的分集内容**。

**验收标准**:

- [ ] AC1: 剧集基础信息
  - 剧集标题(必填,1-100字符)
  - 剧集序号(必填,1-999)
  - 剧集描述(选填,最大500字符)
  - 关联项目(自动关联当前项目)

- [ ] AC2: 剧集状态管理
  - 默认状态"待编写"
  - 支持状态流转(待编写→编写中→待审核→已完成)
  - 状态变更记录时间戳和操作人

- [ ] AC3: 剧集内容结构
  - 自动关联一个空白剧本对象
  - 自动创建分镜管理模块
  - 自动关联角色/场景资产库

- [ ] AC4: 剧集进度跟踪
  - 支持设置里程碑节点
  - 支持查看分镜完成数量
  - 支持查看视频完成数量

**优先级**: P0(MVP核心)

**依赖**: US-001 创建项目

---

## 2. 剧本生产阶段（Script Production）

### US-003: 剧本版本管理

**用户故事**:
作为**编剧**,我想要**创建和管理剧本的多个版本**,以便**保留创作历史并支持回退**。

**验收标准**:

- [ ] AC1: 版本创建
  - 支持从零创建新剧本
  - 支持从现有版本创建新版本(自动复制内容)
  - 版本号自动递增(v1.0→v1.1→v2.0)
  - 支持版本备注(描述本次修改内容)

- [ ] AC2: 版本内容管理
  - 剧本正文支持富文本格式
  - 支持人物列表(姓名/角色定位/简介)
  - 支持场景列表(场景名称/环境描述)
  - 最大字符数:10万字

- [ ] AC3: 版本对比
  - 支持任意两个版本的文本对比
  - 差异内容高亮显示(新增/删除/修改)
  - 支持导出对比报告(PDF格式)

- [ ] AC4: 版本回退
  - 支持将历史版本设为当前版本
  - 回退操作需二次确认
  - 保留完整的版本历史记录

- [ ] AC5: 协作编辑
  - 支持多人同时查看同一剧本
  - 编辑时自动锁定(防止冲突)
  - 锁定超时时间:30分钟

**优先级**: P0(MVP核心)

**依赖**: US-002 创建剧集

---

### US-004: AI辅助剧本分析

**用户故事**:
作为**编剧**,我想要**使用AI分析剧本内容**,以便**快速提取人物关系和场景信息**。

**验收标准**:

- [ ] AC1: 人物关系分析
  - 自动识别剧本中的人物
  - 自动生成人物关系图谱(可视化展示)
  - 支持人工编辑关系图谱
  - 分析结果可保存到剧本元数据

- [ ] AC2: 场景提取
  - 自动识别剧本中的场景
  - 提取场景要素(地点/时间/氛围/关键道具)
  - 生成场景列表并关联到剧集场景库
  - 支持人工调整场景信息

- [ ] AC3: AI任务记录
  - 记录AI调用参数(模型/Token数/耗时)
  - 记录AI生成结果
  - 支持重新生成(使用相同参数)
  - 显示成本估算

- [ ] AC4: 分析失败处理
  - 失败时显示明确错误信息
  - 支持一键重试(最多3次)
  - 超时时间:60秒

**优先级**: P1(重要功能)

**依赖**: US-003 剧本版本管理

**技术备注**:
- 集成文本推理模型(如GPT-4)
- 使用Prompt工程优化分析质量
- 需要设计人物关系图谱数据结构

## 3. 分镜设计阶段（Storyboard Design）

### US-005: 创建分镜

**用户故事**:
作为**导演**,我想要**为剧本创建分镜**,以便**规划每个镜头的视觉呈现**。

**验收标准**:

- [ ] AC1: 分镜基础信息
  - 镜头编号(自动生成,格式:SCENE-序号)
  - 景别(远景/全景/中景/近景/特写)
  - 角度(平视/俯视/仰视/侧视)
  - 运镜方式(固定/推/拉/摇/移/跟)
  - 时长(秒数,1-300秒)

- [ ] AC2: 分镜内容
  - 画面描述(必填,最大500字符)
  - 人物动作(选填,最大300字符)
  - 对白内容(选填,最大500字符)
  - 镜头备注(选填,最大300字符)

- [ ] AC3: Prompt生成
  - 支持AI自动生成图片Prompt
  - 支持人工编辑Prompt
  - 支持从Prompt模板库选择
  - Prompt版本管理

- [ ] AC4: 分镜排序
  - 支持拖拽调整分镜顺序
  - 支持批量调整时长
  - 自动计算总时长

- [ ] AC5: 分镜状态管理
  - 状态:待制作/图片制作中/视频制作中/已完成
  - 状态自动流转
  - 支持手动调整状态

**优先级**: P0(MVP核心)

**依赖**: US-002 创建剧集

---

- [ ] AC2: 分镜调整

  - 支持拆分单个分镜
  - 支持删除/新增分镜
  - 调整后自动重新编号

- [ ] AC3: AI任务记录
  - 记录生成参数
  - 记录Token消耗
  - 支持重新生成

**优先级**: P1(重要功能)

**依赖**: US-003 剧本版本管理, US-005 创建分镜

## 4. 角色与场景资产阶段（Character & Scene Assets）

### US-007: 角色资产管理

**用户故事**:
作为**美术设计师**,我想要**创建和管理角色资产**,以便**保证角色在不同镜头中的一致性**。

**验收标准**:

- [ ] AC1: 角色基础信息
  - 角色名称(必填,1-50字符)
  - 角色定位(主角/配角/路人)
  - 角色简介(必填,最大300字符)
  - 外貌特征(选填,最大500字符)
  - 性格特点(选填,最大300字符)

- [ ] AC2: 角色视觉资产
  - 支持上传参考图(JPG/PNG,最大10MB)
  - 支持AI生成角色图
  - 三视图(正面/侧面/背面)
  - 表情库(至少3种表情)
  - 动作库(可选)

- [ ] AC3: Prompt管理
  - 基础Prompt(角色核心特征)
  - 扩展Prompt(不同场景/情绪变体)
  - Prompt模板(可复用)
  - 支持变量替换(如:{表情}/{动作})

- [ ] AC4: LoRA管理
  - 支持上传自定义LoRA
  - LoRA文件大小限制:100MB
  - LoRA触发词管理
  - LoRA使用统计

- [ ] AC5: 角色引用
  - 分镜中可选择角色
  - 引用角色时自动带入Prompt
  - 支持临时修改Prompt

**优先级**: P0(MVP核心)

**依赖**: US-001 创建项目

## 5. AI图片生产阶段（AI Image Generation）

### US-009: AI图片生成

**用户故事**:
作为**美术设计师**,我想要**通过AI生成分镜图片**,以便**快速产出视觉素材**。

**验收标准**:

- [ ] AC1: 图片生成流程
  - 从分镜选择生图任务
  - 自动加载角色/场景Prompt
  - 支持上传参考图(图生图模式)
  - 支持选择AI模型(默认模型+可选模型列表)
  - 支持设置生成参数(分辨率/风格/采样步数等)

- [ ] AC2: 批量生成
  - 支持批量选择多个分镜
  - 支持队列管理(暂停/取消/重试)
  - 显示生成进度
  - 失败任务自动重试(最多3次)

- [ ] AC3: 结果管理
  - 每次生成展示4张候选图
  - 支持选择/下载/删除
  - 选中的图片自动关联到分镜
  - 支持人工上传图片替代AI生成

- [ ] AC4: 任务记录
  - 记录生成参数(Prompt/模型/Seed/参考图)
  - 记录Token消耗
  - 记录生成耗时
  - 支持使用相同参数重新生成

- [ ] AC5: 成本控制
  - 显示单张图片成本估算
  - 显示项目累计图片成本
  - 超出预算时预警
  - 管理员可设置单项目图片预算

**优先级**: P0(MVP核心)

**依赖**: US-005 创建分镜, US-007 角色资产, US-008 场景资产

**技术备注**:
- 集成图片生成API:POST /v1/images/generations
- 响应时间要求:<30秒
- 需要失败重试机制
- 需要成本统计功能

### US-010: 图片审核

**用户故事**:
作为**审核员**,我想要**审核AI生成的图片**,以便**保证内容质量和合规性**。

**验收标准**:

- [ ] AC1: 审核队列
  - 显示待审核图片列表
  - 按项目/剧集/分镜筛选
  - 显示图片预览+生成参数

- [ ] AC2: 审核操作
  - 通过/驳回/标记问题
  - 驳回时填写原因(必填)
  - 标记问题时选择问题类型(质量/风格/违规)
  - 支持批量审核

- [ ] AC3: 驳回处理
  - 驳回图片自动标记为"待重制"
  - 通知创作者重新制作
  - 显示驳回原因

- [ ] AC4: 审核统计
  - 统计审核通过率
  - 统计常见驳回原因
  - 按时间段导出审核报告

**优先级**: P0(MVP核心)

**依赖**: US-009 AI图片生成

## 6. AI视频生产阶段（AI Video Generation）

### US-011: AI视频生成

**用户故事**:
作为**后期制作人员**,我想要**将分镜图片生成视频**,以便**产出动态内容**。

**验收标准**:

- [ ] AC1: 视频生成流程
  - 从分镜选择生成视频任务
  - 自动加载分镜图片
  - 支持选择AI模型
  - 支持设置运动参数(镜头运动/角色动作)
  - 支持设置时长(1-60秒)

- [ ] AC2: 高级参数
  - 支持设置关键帧(可选)
  - 支持设置运动强度(0-100)
  - 支持设置提示词(描述运动方式)
  - 支持上传参考视频(可选)

- [ ] AC3: 任务队列管理
  - 支持批量提交任务
  - 显示队列状态(排队中/生成中/已完成/失败)
  - 支持取消排队中的任务
  - 失败任务显示错误原因

- [ ] AC4: 结果管理
  - 视频生成完成后自动关联分镜
  - 支持在线预览
  - 支持下载(MP4格式)
  - 支持重新生成

- [ ] AC5: 任务记录
  - 记录生成参数(模型/参数/时长)
  - 记录GPU消耗
  - 记录生成耗时
  - 显示成本估算

**优先级**: P0(MVP核心)

**依赖**: US-009 AI图片生成

**技术备注**:
- 集成视频生成API:POST /v1/videos
- 支持状态查询:GET /agnesapi?video_id=xxx
- 视频生成时间:30-300秒
- 需要异步任务队列机制

## 附录:验收标准编写规范

### 验收标准格式

每个用户故事遵循以下格式:

```
### US-XXX: 功能名称

**用户故事**:
作为[角色],我想要[功能],以便[价值]。

**验收标准**:

- [ ] AC1: 验收条件名称
  - 具体验证点1
  - 具体验证点2
  - 具体验证点3

**优先级**: P0/P1/P2

**依赖**: US-XXX

**技术备注**:
- 技术实现要点
```

### 优先级定义

- **P0(MVP核心)**:第一版本必须实现的核心功能
- **P1(重要功能)**:重要但可延后实现的功能
- **P2(增强功能)**:锦上添花的增强功能

### 验收标准编写原则

1. **可测试性**:每个AC必须有明确的验证方法
2. **完整性**:覆盖正常流程+异常流程+边界情况
3. **独立性**:每个AC独立可验证
4. **明确性**:避免模糊词汇,使用具体数值和条件
